#summary likwid-topology: Print thread and cache topology.




# Introduction #

Modern computers get more and more complicated. They consist of multiple cores
and each core can support multiple hardware threads.  Because cores share
caches and  main memory access it is important to pin threads to dedicated
cores. To understand and use the performance properties of a compute node it is
important to know a machines topology. likwid-topology extracts this
information from the `cpuid` instruction.

likwid-topology reports on:

  * Thread topology: How processor IDs map on physical compute resources
  * Cache topology: How processors share the cache hierarchy
  * Cache properties: Detailed information about all cache levels
  * NUMA topology: NUMA domains and memory sizes

The numbering of the hardware threads as they appear in the Linux kernel depend
on the Bios and can be different even on machines with identical hardware.

# Options #

```
-h       Help message
-v       Version information
-c       list detailed cache parameters
-C       measure processor clock
-o       store output to file (Optional: apply text filter)
-g       graphical output
```

# Usage #

You can get a help message with
```
 likwid-topology -h
```

To print the basic information it is called as the following:
```
 likwid-topology
```

likwid-topology distinguishes processors which are the ids as they appear in
the Linux OS. Thread stands for SMT thread on a core. Core is a physical core.
Socket is what it seems like. Next the sockets are listed with all hardware
thread ids belonging to them. The cache topology is listed for every level.
Here we use the term cache group, which is a subset of hardware
thread ids sharing a cache.

To get more information about the caches use:
```
 likwid-topology -c
```

 likwid-topology -g
